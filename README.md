## 소개

요청이 어디에서 시작되고, 어떤 기준으로 처리되며, 실패했을 때 어떤 기록이 남는지를 중요하게 봅니다.

문제를 바로 덮기보다 먼저 흐름을 나누고, 로그와 재현 가능한 근거를 통해 원인을 좁혀가는 방식을 선호합니다.

## 관심 있는 문제

| 관심사 | 살펴보는 것 |
| --- | --- |
| 서비스 흐름 | 요청이 어디서 시작되고 어디서 멈추는지 |
| 장애 분석 | 실패를 다시 확인하고 재처리할 수 있는지 |
| 운영 가시성 | 로그와 지표가 다음 행동으로 이어지는지 |
| 자동화 | 반복 작업과 실수를 줄일 수 있는지 |
| 개발 도구 | 판단을 방해하지 않고 흐름을 도울 수 있는지 |

## 기록
